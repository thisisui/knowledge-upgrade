## [A designer's guide to opening Xcode for the first time](https://heyneuma.com/xcode-for-designers)

**Summary**
This is a step-by-step onramp for designers (people with no coding background) to build and install a real iPhone app, using Apple's Xcode development software together with Claude Code, an AI coding assistant. Its core premise is that you no longer need to know how to *write* code — only how to *read* it — so the whole guide treats Claude as the engineer and Xcode as just the hands that build and run what Claude produces. It walks through the full path: installing Xcode, signing in with a free Apple ID (no paid developer account needed), the three keyboard shortcuts and handful of screen "landmarks" you actually need, installing three MCP servers (extensions that give Claude real reach into Apple's tools and docs), and finally writing a one-screen app and pushing it onto your own phone. The value is that it removes the intimidation wall that historically stopped non-engineers from shipping anything.

**Key takeaways**
- Treat the AI like an engineer you're briefing, not an image generator: give specific starting context, work in small one-change loops, read the diff for shape (not syntax), keep a plain-English CLAUDE.md project file, and one feature per session.
- Install three MCP servers that make Claude genuinely useful for iOS: XcodeBuildMCP (builds/runs the app), Sosumi (feeds Apple's docs/HIG so it stops inventing non-existent APIs), and GitHub MCP (saves your code).
- Skip the things built for engineers (the SwiftUI Canvas — use the simulator instead), don't fight Apple's defaults, commit every time something works, and "ship the ugly version" to a friend rather than polishing alone.
