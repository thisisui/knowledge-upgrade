## [Why I Don't Use Icon Fonts in Figma](https://www.alicepackarddesign.com/blog/why-i-don-t-use-icon-fonts)

**Summary**
An icon font packs many small pictograms (like a magnifying glass or a trash can) into a single typeface, so designers insert them by typing a character into a text layer. This article argues that, inside the design tool Figma, this approach is worse than storing each icon as an SVG (a scalable vector graphic) saved as a reusable component. The author lists seven concrete problems with icon fonts — they are awkward to insert, force you to memorise icon names with no visual preview, are hard to edit or share across a team, and cannot use multiple colours. The practical recommendation is to keep icons as SVG components, which let designers search visually, swap instances, compare options side by side, and match how developers actually build interfaces.

**Key takeaways**
- SVG components give visual previews, fuzzy search, and one-click instance swapping; icon fonts force you to memorise exact glyph names with no preview.
- Icon fonts can't do multi-colour artwork and are painful to edit (needing font software) or distribute across a team; SVG components avoid all three problems.
- The only real advantage of icon fonts — placing an icon inline within a line of text — is lost in Figma anyway, because mixed fonts in one text layer break text styles and variables.
