## [The Design Engineer Symptom: What a Rising Job Title Reveals](https://uxdesign.cc/the-design-engineer-symptom-what-a-rising-job-title-reveals-850d5e4fd9cc)

**Summary**
This article investigates why job titles like "Design Engineer" and "Product Builder" are appearing more and more in job listings, and what this trend actually means for how software is built. Traditionally, designers and developers are two separate roles: designers decide what the product should look like and how it should work, then hand their plans over to developers who write the code to make it real. The "design engineer" is someone — or a way of working — that removes this handoff by having one person (or a designer using AI coding tools) take an idea all the way from visual concept to working, shipped product. The article includes real examples from companies where designers were given access to AI coding tools and, as a result, independently fixed user interface problems that would previously have required a developer. The author concludes that AI did not invent this role, but it has made it much more accessible by allowing designers to generate working code without having to learn programming from scratch.

**Key takeaways**
- Companies like Alan (a French health app) gave their designers access to AI-assisted coding tools and saw over 350 code changes merged in a single quarter, with most designers actively contributing to the codebase — work that previously required a developer to execute.
- The hardest part of designers writing code is not generating the code itself, but understanding how to verify that the change works correctly and does not break anything else. Testing and reviewing code changes turned out to be half the challenge in real-world experiments.
- This shift toward designers writing code is currently concentrated in smaller companies and startups. Larger organizations and industries with strict regulations (like banking or healthcare) still require clearly separated roles, so this change will not happen everywhere at the same pace.

*Relevant for teams deciding how to restructure design-dev collaboration around AI tools — the real-world case studies are concrete, but the trend is still early-stage.*
