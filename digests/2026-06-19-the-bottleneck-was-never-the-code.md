## [The bottleneck was never the code](https://www.thetypicalset.com/blog/thoughts-on-coding-agents)

**Summary**
Coding agents make individual code production fast and cheap — but that was never the real constraint in software delivery. The author argues that software is fundamentally the output of humans negotiating with each other, and as code generation becomes effortless, the bottleneck shifts visibly to organizational alignment: clear specifications, coherent roadmaps, the discipline to say no to feasible but low-value features, and the tacit knowledge that teams accumulate but rarely write down. Drawing on Brooks (1975) and Weinberg (1971), the piece warns that faster code output will amplify organizational dysfunction rather than cure it. The emerging competitive advantage is not an AI that writes faster code, but systems that extract and codify the implicit organizational context that agents need to work correctly.

**Key takeaways**
- Code was never the bottleneck — alignment, specs, and focus discipline were; AI just makes that visible
- "Context as currency": organizational knowledge that was never written down now directly limits what agents can produce
- Documentation is infrastructure — teams that invest in making implicit knowledge explicit will compound their advantage
