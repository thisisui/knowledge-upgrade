## [The interface is no longer the product](https://blog.mozilla.ai/the-interface-is-no-longer-the-product/)

**Summary**
For decades, software was designed around two human interaction patterns: issuing commands (clicking, typing) and manipulating visual representations (dragging, resizing). AI agents do not need either. They need structured state they can read, reason about, and rewrite — and the author argues this means the software categories we recognise (slides, spreadsheets, documents) are "accidents of interface history," not fundamental structures. The real product is the underlying information; the interface is just a projection of it. Applications that are rebuilt natively for agents will expose structured representations directly, with human-readable views as an output layer, rather than building agents on top of legacy interfaces. The scarce resource shifts from the rendering layer to whatever system owns the structured understanding of what an artifact actually is.

**Key takeaways**
- Software categories bundle data model + renderer + editor + collaboration — agents only need the data model; the rest is overhead built for humans.
- "A chatbot next to a legacy app is not the same thing as an agent-native application" — the bridge and the destination are different things.
- The friction points where agents struggle with existing tools are structural signals about which categories are most ripe for reinvention.
